

Task Manager - это полнофункциональное веб-приложение для управления личными задачами. Проект создан в рамках стажёрского задания и демонстрирует навыки full-stack разработки.

/screenshots/main-view.png

✨ Возможности

- ✅ Просмотр всех задач** — список всех созданных задач
- ➕ Добавление новых задач** — простая форма для быстрого добавления
- ✔️ Отметка задач как выполненных** — переключение статуса через чекбокс
- 🗑️ Удаление задач** — ненужные задачи можно удалить
- 🔄 Автоматическое обновление** — интерфейс обновляется без перезагрузки страницы
- 📱 Адаптивный дизайн** — удобно работать на любых устройствах

🛠️ Стек технологий

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| Backend | Node.js + Express | Серверная логика и API |
| Frontend | React + Vite | Пользовательский интерфейс |
| База данных | PostgreSQL | Хранение задач |
| HTTP-клиент | Fetch API | Запросы к серверу |
| Стилизация | CSS | Внешний вид приложения |
| Контроль версий | Git + GitHub | Хранение кода |

🚀 Быстрый старт

Предварительные требования

Перед началом убедитесь, что у вас установлены:

- [Node.js](https://nodejs.org/) (версия 18 или выше)
- [PostgreSQL](https://www.postgresql.org/) (версия 14 или выше)
- [Git](https://git-scm.com/)
- Редактор кода (рекомендуется VS Code)

1. Клонирование репозитория

bash
git clone https://github.com/Ememdencik137/task-manager.git
cd task-manager


2. Настройка базы данных

Способ 1: Через командную строку

Подключитесь к PostgreSQL и выполните:

sql
CREATE DATABASE task_manager;
\c task_manager;

CREATE TABLE tasks (
    id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    completed BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


Способ 2: Через pgAdmin

1. Создайте базу данных с именем `task_manager`
2. Откройте Query Tool и выполните SQL-запрос выше

3. Запуск backend

bash
Перейдите в папку backend
cd backend

Установите зависимости
npm install


Откройте файл `backend/server.js` и укажите свой пароль от PostgreSQL:

javascript
const pool = new Pool({
    user: 'postgres',
    host: 'localhost',
    database: 'task_manager',
    password: 'qwerty6767',  
    port: 5432,
});


Запустите сервер:

bash
npm run dev


Сервер будет доступен по адресу: `http://localhost:5000`

4. Запуск frontend

Откройте новый терминал (не останавливая сервер) и выполните:

bash
Перейдите в папку frontend
cd frontend

Установите зависимости
npm install

Запустите React-приложение
npm run dev

Приложение откроется по адресу: `http://localhost:5173`

📡 API Endpoints

| Метод | Endpoint | Описание | Пример ответа |
|-------|----------|----------|---------------|
| GET | `/tasks` | Получить все задачи | `[{ "id": 1, "title": "Купить молоко", "completed": false, "created_at": "2024-01-01T12:00:00Z" }]` |
| POST | `/tasks` | Создать новую задачу | Отправить: `{ "title": "Новая задача" }` |
| PUT | `/tasks/:id` | Обновить статус задачи | Отправить: `{ "completed": true }` |
| DELETE | `/tasks/:id` | Удалить задачу | `{ "message": "Задача удалена" }` |

🎯 Примеры использования

Добавление задачи
1. Введите текст задачи в поле ввода
2. Нажмите кнопку "Добавить" или Enter
3. Задача появится в списке

Выполнение задачи
- Нажмите на чекбокс слева от задачи
- Задача зачеркнется — статус изменится на "выполнено"

Удаление задачи
- Нажмите красную кнопку "Удалить" справа от задачи
- Задача исчезнет из списка

⚠️ Важные замечания

1. Безопасность: Никогда не коммитьте файлы с реальными паролями! Используйте переменные окружения для продакшена.

2. Зависимости: Папки `node_modules` не включены в репозиторий. После клонирования всегда выполняйте `npm install` в обеих папках.

3. Порты: Убедитесь, что порты 5000 (backend) и 5173 (frontend) свободны.
